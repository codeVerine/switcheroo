# Switcheroo 0.1.1

Released on 2026-05-13.

## Install

- App: download `Switcheroo-0.1.1-macos-arm64.dmg`
- CLI: download `switcheroo-0.1.1-macos-arm64.tar.gz`

## Highlights

- Identity-aware snapshot refresh: importing/finalizing a login now refreshes an existing matching account snapshot instead of silently creating duplicates.
- Safer, quieter background sync: the menu bar app only polls near token refresh time, otherwise it schedules a later recheck.
- Clearer UX: import-first copy, compact status/error banners, and expiry shown only when it matters.

## Detailed Changes

### Identity-aware snapshot upserts and de-duplication

- Switcheroo derives a best-effort identity key from the active `auth.json` (preferring `account_id`, with `user_id`/email fallback) and uses it to match existing configured accounts.
- When a match is found, Switcheroo updates the existing Keychain snapshot in-place and (when appropriate) corrects the configured active account id to the matched account.
- Older configs that predate the identity key format are handled by backfilling identity keys from stored Keychain snapshots to keep matching stable as identity semantics evolve.

### Menu bar auto-sync window + re-login detection

- Menu bar now runs sync on launch, before switching accounts, and then only polls every ~15 seconds when the active access token is within **2 days + 5 minutes** of expiry.
- Outside the refresh window, the app schedules a later recheck rather than continuously reading credentials.
- When the active auth is unreadable or cannot be confidently matched to an existing configured account identity, the UI shows a compact `Re-login required.` banner and background polling is disabled (with a low-frequency recheck to allow the banner to clear after login).

### CLI behavior + messaging

- CLI opportunistically syncs once per invocation (best-effort) and avoids spamming `Re-login required.` on fresh/no-account setups.
- `switcheroo import-current` / `switcheroo add` now report when the action refreshed an existing account instead of always claiming a new import/add.

### UX polish

- "Import logged-in account" replaces ambiguous "Sync current account" wording and uses provider-aware copy in the empty state.
- Expiry display is only shown when relevant (expired, or under 24 hours remaining), with clearer tooltips.
- Reduced tooltip hover delay for snappier interactions.

### Release automation and versioning

- Release workflow now generates release notes from git history (with install instructions) and attaches standard artifacts.
- App bundling script stamps the app bundle version from `VERSION`/tag, and the menu bar UI reads the bundle version instead of hardcoding it.

## Notes / Compatibility

- Switcheroo still does not refresh tokens; it only snapshots whatever the official Codex login/refresh writes to `~/.codex/auth.json`.
- Background sync will never create new accounts; it only updates an existing matching account snapshot.
